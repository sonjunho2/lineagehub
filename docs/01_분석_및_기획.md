# 참고 사이트 분석 및 반영 방향

## 1) 공식 PlayNC 정보 구조
- 아이템 / 마법 / 몬스터 3축 정보 구성이 확인됨.
- 따라서 본 프로젝트도 공식 정보 구조를 기준으로 `아이템 DB`, `몬스터 DB`, 추후 확장용 `마법 DB`를 분리 설계.

## 2) GAMEBIT 시세 구조
- 서버별 현재가, 시간 기준 시세, 거래량, 등락률, 캔들 차트, 노이즈 필터링, 면책 고지가 핵심.
- 본 프로젝트는 서버별 최신가 + 24시간 시계열 차트 + 거래량 + 면책 안내 구조를 반영.
- 사용자가 원치 않은 `네임드 순위`와 `최근 거래완료`는 제외.

## 3) D4GG 구조
- 홈, 서버구도, 거래소, 맵정보, 스킨/혈마크, DB, 색상코드 메뉴 구조가 노출됨.
- 본 프로젝트는 탐색성을 위해 홈 / DB / 시세 / 커뮤니티 / 러쉬 / 관리자 중심으로 단순화.
- 스킨/혈마크와 색상코드는 후속 확장 메뉴로 반영 가능하게 설계.

## 4) adena.kr 강화 도구 구조
- Single/Multi Enchant, 주문서 타입, 시도/성공/파괴/예상 비용 통계, 시세 연동형 계산 기능이 핵심.
- 본 프로젝트는 `포인트 차감형 러쉬 시뮬레이터`, 주문서 단가 입력, 실행 통계, 실행 이력 저장으로 반영.

## 5) Inven 커뮤니티 구조
- 자유게시판, 질문답변, 패치소식, 스샷, 혈맹모집, 거래게시판, 클래스/서버 게시판 등 게시판 세분화가 핵심.
- 본 프로젝트는 초기 운영 효율을 위해 `공지`, `팁`, `시세토론`, `혈맹모집`, `질문답변` 5개 보드로 시작.

## 중요한 운영 판단
참고 사이트의 데이터/본문/이미지 전체를 그대로 복제하면 저작권·이용약관 문제가 발생할 수 있으므로, 본 샘플은 **기능 구조와 정보 아키텍처를 반영한 독립 구현체**다. 실제 운영 시에는
1. 직접 수집·작성한 데이터,
2. 사용 허가를 받은 데이터,
3. 공식 API/허용된 수집 경로
만 연결해야 한다.
